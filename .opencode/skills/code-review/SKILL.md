# 代码审查技能

对代码进行审查，确保符合项目规范。

## 审查维度

### 1. 架构合规性

- [ ] 是否遵循 Clean Architecture 分层
- [ ] 依赖方向是否正确（上层依赖下层）
- [ ] Domain 层是否有 Android 依赖
- [ ] Feature 层是否直接使用了 CameraX API
- [ ] SDK 层是否依赖了其他项目模块

### 2. 代码风格

- [ ] 是否有 KDoc 注释
- [ ] 命名是否符合规范（大驼峰类名、小驼峰函数名）
- [ ] 是否有魔法数字
- [ ] 是否有未使用的导入
- [ ] 缩进是否正确（4 空格）

### 3. 线程安全

- [ ] OpenGL 操作是否在渲染线程
- [ ] 跨线程通信是否使用 @Volatile 或 Handler
- [ ] IO 操作是否在 Dispatchers.IO
- [ ] UI 更新是否在主线程

### 4. 资源管理

- [ ] SurfaceTexture 是否正确释放
- [ ] Bitmap 是否正确回收
- [ ] Fragment 的 _binding 是否在 onDestroyView 置空
- [ ] 相机资源是否在 onDestroyView 释放

### 5. 错误处理

- [ ] 是否有 try-catch 处理异常
- [ ] 是否有日志记录错误
- [ ] 是否有用户友好的错误提示

### 6. 性能优化

- [ ] 是否有内存泄漏风险
- [ ] 是否有不必要的对象创建
- [ ] 是否有阻塞主线程的操作

## 输出格式

```markdown
## 代码审查报告

### 通过项
- ✅ 架构合规
- ✅ 代码风格良好

### 问题项
- ❌ [严重] 发现问题描述
  - 文件：xxx.kt:123
  - 建议：修复建议

### 改进建议
- 💡 建议描述
```
